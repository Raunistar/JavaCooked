-->Selection sort => The core idea of the selection sort is to find the min and swap with same element.

Core Concept
Find the minimum element in the unsorted portion and swap it with the beginning of the unsorted portion.

Algorithm Steps
Start with the first element as the minimum

Compare with all remaining elements to find the actual minimum

Swap the found minimum with the first element

Move boundary between sorted and unsorted portions one element to the right

Dry Run Example:

Array: [12, 4, 3, 7, 32]

Pass 1:
  Assume min at index 0 (12)
  Compare with [4,3,7,32] → min=3 at index 2
  Swap arr[0] with arr[2] → [3, 4, 12, 7, 32]

Pass 2:
  Assume min at index 1 (4)
  Compare with [12,7,32] → min=4 at index 1
  No swap needed → [3, 4, 12, 7, 32]

Pass 3:
  Assume min at index 2 (12)
  Compare with [7,32] → min=7 at index 3
  Swap arr[2] with arr[3] → [3, 4, 7, 12, 32]

Pass 4:
  Assume min at index 3 (12)
  Compare with [32] → min=12 at index 3
  No swap needed → [3, 4, 7, 12, 32]

Sorted Array: [3, 4, 7, 12, 32]

Key Points ->
In-place sorting: Doesn't require extra memory

Unstable sort: Equal elements may not maintain original order

Time Complexity: Always O(n²) - even if array is sorted

Space Complexity: O(1)

Comparisons: n(n-1)/2 comparisons

Swaps: Maximum n-1 swaps


-------------------------------------------------------------------------------------------------------Core Concept
                                                                                                       Repeatedly compare adjacent elements and swap them if they're in wrong order. Largest elements "bubble up" to their correct position.

                                                                                                       Algorithm Steps
                                                                                                       Compare first two elements, swap if needed

                                                                                                       Move to next pair, compare and swap

                                                                                                       Continue until end of array

                                                                                                       Repeat process for remaining unsorted portion

                                                                                                       Dry Run Example
                                                                                                       text
                                                                                                       Array: [12, 2, 11, 23, 4]

                                                                                                       Pass 1 (Compare all adjacent pairs):
                                                                                                         (12,2) → swap → [2, 12, 11, 23, 4]
                                                                                                         (12,11) → swap → [2, 11, 12, 23, 4]
                                                                                                         (12,23) → no swap → [2, 11, 12, 23, 4]
                                                                                                         (23,4) → swap → [2, 11, 12, 4, 23]
                                                                                                         Largest element (23) now at correct position

                                                                                                       Pass 2 (Ignore last element):
                                                                                                         (2,11) → no swap → [2, 11, 12, 4, 23]
                                                                                                         (11,12) → no swap → [2, 11, 12, 4, 23]
                                                                                                         (12,4) → swap → [2, 11, 4, 12, 23]
                                                                                                         Second largest (12) at correct position

                                                                                                       Pass 3 (Ignore last 2 elements):
                                                                                                         (2,11) → no swap → [2, 11, 4, 12, 23]
                                                                                                         (11,4) → swap → [2, 4, 11, 12, 23]
                                                                                                         Third largest (11) at correct position

                                                                                                       Pass 4 (Ignore last 3 elements):
                                                                                                         (2,4) → no swap → [2, 4, 11, 12, 23]
                                                                                                         Array is sorted!

 for this algo T.C = O(N)  -best case only if the given array is already sorted
                     O(N^2) - worst and avg and most common

-->Insertion sort => Insertion sort is a simple sorting algorithm that works like sorting playing cards in your hand: you build a sorted section one element at a time by inserting each new element into its correct position.

🃏 How Insertion Sort Works
Step 1: Assume the first element is already sorted.

Step 2: Pick the next element (called the key).

Step 3: Compare the key with elements in the sorted part.

Step 4: Shift larger elements one position to the right.

Step 5: Insert the key into its correct spot.

Step 6: Repeat until all elements are sorted.

👉 Think of it like arranging cards: you hold a sorted hand and keep inserting the next card where it belongs.

⚙️ Example Walkthrough
Suppose we want to sort [5, 3, 4, 1, 2]:

Start with [5] (sorted).

Insert 3: compare with 5, place before → [3, 5].

Insert 4: compare with 5, place before → [3, 4, 5].

Insert 1: place at the beginning → [1, 3, 4, 5].

Insert 2: place between 1 and 3 → [1, 2, 3, 4, 5].

for this algo T.C = O(N)  -best case only if the given array is already sorted
                     O(N^2) - worst and avg and most common
